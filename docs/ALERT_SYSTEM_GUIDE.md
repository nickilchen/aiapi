# 🔔 告警通知系统使用指南

## 📖 简介

告警通知系统是 AIAPI 管理控制台的核心功能之一，提供实时监控和多渠道告警通知能力，帮助您及时发现和处理系统异常。

## ✨ 核心功能

### 1. 智能监控

- **提供商健康状态监控**：实时监控提供商账号的健康率，当健康率低于阈值时触发告警
- **API 错误率监控**：统计分析 API 请求的错误率，超过阈值时及时告警
- **配额使用监控**：追踪 API 配额使用情况，避免配额耗尽影响服务

### 2. 多渠道通知

- **浏览器实时通知**：通过 SSE 和 Notification API 实现浏览器实时推送
- **企业微信通知**：支持企业微信群机器人，可 @指定成员
- **自定义 Webhook**：支持发送到任意 HTTP 端点，灵活集成第三方系统

### 3. 告警管理

- **告警历史记录**：完整记录所有告警信息，支持查询和筛选
- **告警确认机制**：支持手动确认告警，避免重复处理
- **冷却时间控制**：防止告警风暴，同一告警在冷却期内不会重复发送

## 🚀 快速开始

### 1. 访问告警页面

1. 打开浏览器访问：`http://localhost:3000`
2. 登录管理控制台
3. 点击侧边栏的"告警通知"菜单

### 2. 启用告警系统

在告警配置面板顶部，开启"启用告警系统"开关。

### 3. 配置告警规则

#### 提供商健康状态

- **健康率阈值**：设置健康率百分比（建议 50%）
- **冷却时间**：设置告警间隔时间（建议 5 分钟）

#### 错误率监控

- **错误率阈值**：设置错误率百分比（建议 10%）
- **时间窗口**：设置统计时间范围（建议 5 分钟）
- **冷却时间**：设置告警间隔时间（建议 5 分钟）

#### 配额使用监控

- **配额阈值**：设置配额使用百分比（建议 90%）
- **冷却时间**：设置告警间隔时间（建议 1 小时）

### 4. 配置通知方式

#### 浏览器通知

1. 开启"浏览器通知"开关
2. 点击"请求通知权限"按钮
3. 在浏览器弹窗中允许通知

#### 企业微信通知

1. 在企业微信群中添加机器人，获取 Webhook URL
2. 开启"企业微信通知"开关
3. 填写 Webhook URL
4. （可选）填写需要 @的成员 userid 或手机号
5. 保存配置

**获取企业微信 Webhook URL：**
- 在企业微信群中，点击右上角 "..." → "群机器人" → "添加机器人"
- 复制 Webhook 地址，格式如：`https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxx`

**@成员说明：**
- userid：企业微信成员的用户ID，多个用逗号分隔
- 手机号：成员的手机号，多个用逗号分隔
- 使用 `@all` 可以提醒所有人

#### 自定义 Webhook

1. 开启"自定义 Webhook"开关
2. 填写 Webhook URL
3. 选择请求方法（POST/PUT）
4. （可选）配置自定义请求头（JSON 格式）
5. 保存配置

### 5. 测试告警

点击页面右上角的"测试告警"按钮，验证配置是否正确。

## 📊 监控指标说明

### 提供商健康状态

```
监控频率：每分钟
触发条件：健康率 < 阈值
告警内容：
  - 提供商类型
  - 健康账号数
  - 总账号数
  - 健康率百分比
```

### 错误率监控

```
监控频率：每分钟
统计窗口：可配置（默认 5 分钟）
触发条件：错误率 > 阈值
告警内容：
  - 提供商类型
  - 错误次数
  - 总请求数
  - 错误率百分比
```

### 配额使用监控

```
监控频率：每分钟
触发条件：配额使用 > 阈值
告警内容：
  - 提供商类型
  - 已用配额
  - 总配额
  - 使用率百分比
```

## 📝 告警历史管理

### 查看告警历史

1. 在"告警历史"面板中查看所有告警记录
2. 使用类型和严重程度筛选器过滤告警
3. 点击分页按钮浏览更多记录

### 确认告警

1. 找到需要确认的告警
2. 点击"确认"按钮
3. 已确认的告警会显示为灰色

### 清除历史

点击页面右上角的"清除历史"按钮，可以清空所有告警历史记录。

## 🎯 最佳实践

### 1. 合理设置阈值

- **健康率阈值**：建议设置为 50%，确保至少一半的账号可用
- **错误率阈值**：建议设置为 10%，避免过于敏感
- **配额阈值**：建议设置为 90%，留出缓冲时间

### 2. 配置冷却时间

- **提供商健康**：5-10 分钟，避免频繁告警
- **错误率**：5-10 分钟，给系统恢复时间
- **配额使用**：1 小时，配额变化较慢

### 3. 选择合适的通知方式

- **浏览器通知**：适合实时监控，需要保持浏览器打开
- **企业微信**：适合团队协作，支持 @成员
- **Webhook**：适合集成到现有监控系统

### 4. 定期检查告警历史

- 每天查看告警历史，了解系统运行状况
- 分析告警趋势，优化系统配置
- 及时确认已处理的告警

## ❓ 常见问题

### Q1: 浏览器通知不显示？

**A:** 请检查：
1. 是否已授予浏览器通知权限
2. 浏览器是否支持 Notification API
3. 是否开启了浏览器的"请勿打扰"模式

### Q2: 企业微信通知发送失败？

**A:** 请检查：
1. Webhook URL 是否正确
2. 机器人是否被移除或禁用
3. 网络是否可以访问企业微信 API

### Q3: 告警太频繁怎么办？

**A:** 可以：
1. 调高告警阈值
2. 增加冷却时间
3. 优化系统配置，减少错误

### Q4: 如何测试告警配置？

**A:** 点击"测试告警"按钮，系统会发送一条测试告警到所有已启用的通知渠道。
